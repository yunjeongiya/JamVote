## 1. JamVote 프로젝트 초기 셋업 및 구현
**완료 시각**: 20:50 KST

### 수정 목적 및 내용

#### 프로젝트 초기화
- **목적**: 밴드 합주곡 선정을 위한 투표 서비스 구축
- **기술 스택 결정**: Next.js + TypeScript + Tailwind CSS
- **프로젝트 생성**: `npx create-next-app@latest jamvote` 명령으로 Next.js 15.5.2 프로젝트 생성

#### 메인 페이지 구현 (app/page.tsx)
- **목적**: 모든 핵심 기능을 단일 페이지에 통합한 SPA 구현
- **구현 기능**:
  1. **곡 제안 기능**: 제목, 아티스트, YouTube URL 입력 폼
  2. **투표 시스템**: 사용자별 중복 방지, 투표/취소 토글
  3. **YouTube 임베드**: URL 파싱 후 iframe 플레이어 자동 생성
  4. **코멘트 기능**: 각 곡별 댓글 작성 및 표시
  5. **자동 정렬**: 투표 수 기준 내림차순 정렬

#### 데이터 저장 방식
- **LocalStorage 활용**: 
  - `jamvote-songs`: 곡 목록 데이터
  - `jamvote-username`: 사용자 이름
- **데이터 구조 정의**:
  - Song 인터페이스: id, title, artist, youtubeUrl, votes, votedBy[], comments[], suggestedBy, createdAt
  - Comment 인터페이스: id, author, text, createdAt

#### UI/UX 구현
- **다크모드 지원**: Tailwind CSS 다크모드 클래스 적용
- **반응형 디자인**: 모바일/데스크톱 대응
- **사용자 경험**: 첫 액션 시 이름 입력, 이후 자동 저장
