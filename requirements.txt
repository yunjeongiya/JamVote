밴드에서 합주곡을 정하는 걸 돕는 간단한 모바일 기반 웹서비스.
사용자들은 모바일 환경에서 접근할 가능성이 높다.
현재 밴드 합주곡 선정 시 단순 메신저 단톡방에 직접 곡을 올리고 이야기하고 투표하는 식인데, 이 때 사람마다 제목, 가수를 정확히 적지 않고
사람마다 올리는 형식이 다르고 가끔은 떠올리는 버전이 다르며 모르는 곡이면 매 번 검색해서 들어봐야 한다.
이 문제를 간단한 웹서비스로 해결하려 한다.
보통 합주곡을 정할 땐 서로 모르기 때문에 이름만이 아니라 세션 정보를 프로필에 같이 넣어 주어 소통 편의성을 높인다.
사이트 내에서 바로 유튜브 검색, 재생, 의견 공유, 세션 난이도 확인, 곡 투표 등을 할 수 있어 편의성을 높인다.
일회성 서비스이므로 복잡한 회원가입은 생략하고 간단하게 사용할 수 있도록 한다.

0. 소통
- 모든 화면에는 우측 하단 말풍선 플로팅 버튼이 있다.
- 말풍선 버튼을 누르면 모달 창이 뜨며, 별점, 평가, 문의사항 등을 입력할 수 있다.
- 모달 창에는 개발자 정보와 연락처도 함께 표시된다.
- 입력된 내용은 우선은 자체 DB에 저장하여 개발자가 직접 확인하고, 차후 email로 전송하거나 다른 메신저를 연동한다.

1. 방 생성 & 접속
1.1 기능
- When2Meet과 같이 한 사람이 방의 개념인 Jam 을 만들면, 링크와 방 아이디가 생성된다.
- 자체 링크 혹은 메인 화면에 방 아이디를 입력해 다른 사람이 방에 입장할 수 있다.
- 방 유효기한이 지나면 자동으로 방이 삭제되며, 이는 데이터베이스 공간 확보를 위해 필요하다.

1.2 화면 구성
- 화면 중앙에는 서비스 이름, 아이콘, 간략한 소개가 있다.
- 화면 하단에는 방 아이디 필드와 접속하기 버튼이 있다.
- 접속 버튼 라인 하단에는 방 이름 필드(필수x), 방 설명 필드 (필수x), 방 유효기한 필드(자동으로 최초값 1달 설정, 수정 가능), 방 생성 버튼이 있다.
    - 방 이름을 넣지 않으면 자동으로 방 아이디가 방 이름으로 설정된다.
    - 방 설명을 넣지 않으면 자동으로 빈 칸으로 설정된다.
    - 방 유효기한을 넣지 않으면 자동으로 1달로 설정된다.
- 방 생성 버튼을 누르면 하단에 새로 생성된 방 링크와 복사 버튼이 생성되고, 방 아이디 필드에 방 아이디가 입력된다.
- 사용자가 복사 버튼을 누르지 않아도 우선 링크 생성시 자동으로 클립보드에 복사되며 알림이 뜬다.
- 방 접속하기 버튼을 누르면 방 가입 & 로그인 화면으로 이동한다.

2. 방 가입 & 로그인
2.1 기능
- 최초 접속일 경우 간단하게 프로필을 생성한다.
- 프로필 정보로는 이름, 비밀번호(선택), 세션이 있다.
    - 이름은 각 방에서 유니크한 ID 역할을 하므로 중복 검사를 한다.
    - 비밀번호는 미설정시 비밀번호 없이 입장 가능하다. 
    - 다른 사람이 이름만 입력해 들어올 수도 있으나 서비스 특성 상 보안성이 중요하지 않으므로 편의성을 위해 비밀번호 없이 입장 가능하도록 한다.
    - 0~n개의 세션을 선택할 수 있다.
- 최초 접속이 아닐 경우 이름과 비밀번호를 입력해 로그인한다.
- 프로필 생성 혹은 로그인 완료 후 방 진행 화면으로 이동한다.
2.2 화면 구성
- 화면 중앙에는 서비스 이름, 아이콘, 간략한 소개가 있다.
- 화면 하단에는 이름 필드, 비밀번호 필드, 로그인 버튼이 있다.
- 프로필 생성 버튼을 누르면 프로필 생성 화면으로 이동한다.
- 프로필 생성 화면에는 이름 필드, 비밀번호 필드, 세션 필드, 프로필 생성 버튼(2)이 있다.
- 로그인 버튼 혹은 프로필 생성 버튼(2)을 누르면 프로필 생성 완료 후 방 진행 화면으로 이동한다.

3. 방 진행
3.1 기능
3.1.1 곡 제안
- 방에 입장하면 유튜브 검색이 연동된 검색창을 통해 곡을 입력할 수 있다.
- 검색창에 곡을 입력하면 유튜브 검색 결과가 나오고, 그 중 하나를 선택하면 곡 정보가 입력된다.
- 곡 정보로는 제목, 가수, 재생시간, 유튜브 링크, 장르, 필요 세션 및 세션별 난이도가 있다.
- 최초에는 유튜브 영상 제목과 영상 올린 사람 이름, 재생시간으로 자동으로 제목과 가수, 재생시간이 입력되나, 사용자가 자유롭게 수정 가능하다.
- 장르, 필요 세션, 세션별 난이도의 경우 잘 모를 경우 비워두고 다른 사람들이 입력하도록 허용할 수 있다.
- 이후 기술적으로 가능하다면, 제목, 가수, 재생시간 외의 정보들도 웹사이트가 직접 판단해 자동으로 채워주도록 할 수 있다.
- 이는 AI를 활용하거나, 사이트가 DB를 구축하여 데이터를 활용할 수 있다.
3.1.2 곡 투표
- 곡 리스트의 각 컴포넌트에는 투표 버튼이 있으며, 클릭 시 바로 투표되고 숫자와 투표한 사람 정보가 반영된다.
- 투표는 기명투표로, 투표자 이름과 세션 정보가 노출된다.
- 투표로는 좋아요와 불가능 두 가지로 구성되며, 불가능 클릭 시 댓글로 이유 작성을 유도하는 모달이 뜬다.

3.2 화면 구성
3.2.1 상단 바
- 화면 상단에는 좌측에 방 유효기한 디데이, 중앙에 방 이름, 우측에 본인 세션 아이콘, 이름이 표시된다.
- 방 유효기한은 아무나 클릭 시 수정 가능하며, 하루 내일 경우 빨간색으로 표시한다.
- 본인 세션 아이콘과 이름 영역을 클릭하면 프로필 수정 화면으로 이동한다.
- 보안보다 편의성이 중요하므로 비밀번호 재검증 없이 수정을 허용한다.
- 이름은 방 내에서 중복되지 않을 경우 수정 가능하다.
- 세션도 수정 가능하다.
3.2.2 곡 검색
- 화면에는 리스트뷰로 여태 제안된 곡들이 뜨고, 목록의 최상단에는 새로운 곡 제안을 위한 검색창이 있다.
- 디스코드 포럼처럼 입력값이 현재 리스트에 있는지 먼저 검색되고, 그 밑에 유튜브 검색 결과가 나온다.
- 유튜브 검색 결과 클릭 시 곡 정보 입력 화면으로 이동한다.
3.2.3 곡 정보 입력
- 곡 입력 화면에는 제목, 가수, 재생시간이 맨 윗줄에 뜨고, 그 밑에 유튜브 임베드, 그 밑에 장르, 필요 세션, 세션별 난이도가 있다.
- 유튜브 임베드는 바로 재생해볼 수 있으며, 원하는 곡이 아니었을 경우 뒤로가기로 바로 나와서 리스트로 돌아갈 수 있다.
- 제목, 가수, 재생시간은 자동으로 채워져 있으나 자유롭게 수정 가능하다.
- 장르, 필요 세션, 세션별 난이도는 잘 모를 경우 비워두고 다른 사람들이 입력하도록 허용할 수 있다. 기본값은 허용으로 체크되어 있다.
3.2.4 곡 리스트
- 리스트는 투표 수 내림차순, 작성일 내림차순으로 정렬된다.
- 리스트뷰 각 컴포넌트에는 펼쳐보기 버튼, 곡 제목, 가수, 필요 세션 아이콘, 투표 버튼이 있다.
- 투표 버튼은 좋아요와 불가능 두 가지로 구성되며, 본인이 투표한 결과가 표시되고, 투표 수가 표시된다.
- 펼쳐보기 버튼을 누르면 곡 정보, 투표 결과, 댓글이 펼쳐지고, 곡 정보에는 장르, 필요 세션, 세션별 난이도, 유튜브 임베드가 있다.
- 투표 결과로는 투표한 사람들의 이름과 세션 아이콘이 표시된다.
- 리스트뷰를 유지하며 펼쳐보기 한 곡의 유튜브 임베드로 영상을 바로 재생해 볼 수 있다.
- 펼쳐보기 상태에서는 자유롭게 댓글을 달 수 있으며, 댓글은 작성자 이름, 세션 아이콘, 댓글 내용, 작성일이 표시된다.
- 작성자가 허용한 경우 아무나 곡 정보를 수정할 수 있다.
